1.包名：全小写；英文名尽量全拼或者是缩写成4个或5个字母；
2.表名是下划线；
3.数据库字段：英文名尽量全拼；
4.主键必须是ID，是整形；
5.类名：首字母大写，驼峰形式(UserRole)；
6.注释：（1）类注释：要求至少写出内容说明、创建者、创建时间和特别注意事项等内容
				（2）方法说明注释：每个方法都应有适当的说明，位于方法声明之前，包括：说明，参数说明、异常说明、返回值说明和特别说明等
				（3）分支语句的注释：在分支语句的下一行进行注释，简单描述该分支存在的意义；
						示例：if(1=1){
										//如果1与1相等时
										code
									}else{
										//如果1与1不相等时
										code
									}
7.一个方法超过一屏后必须拆分为多个小方法；
8.黄色叹号的js提示、java 包提示等必须清楚掉！
9.不要外键约束；自己写删除、级联。
10.使用大写命名用public static final类型变量（容易区分常量和本地变量）；
11.提交到svn中的代码，js中的alert，java代码中的system.out.println()必须注释或删除；


后台html表单注意事项：
1.添加校验规则，参考常用示例-单表模型-校验；
2.必填项红色星号应该在文字的左边；
3.下拉选择框在编辑的时候应该要选对了；
4.单选框和复选框的样式需要调整后才能正常显示和点击；
5.图片上传和文件上传参考文章里的代码；
6.排列顺序这种需要数字的默认为0，数字越大排列越靠前，大多数排列应该都是orderby 排序字段 desc,id desc;
